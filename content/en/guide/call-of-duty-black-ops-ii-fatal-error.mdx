When launching Call of Duty: Black Ops II, the notorious "Fatal Error: Unhandled Exception Caught" crash often points to missing shared libraries or conflicts with the modern Steam client overlay. Before adjusting deep system files, run the standard Steam verification process and toggle the client beta configuration.

| Fix Action | Target Issue | Expected Outcome |
| :--- | :--- | :--- |
| **Verify Integrity** | Corrupted or missing local game files | Downloads missing 3.4 MB patch |
| **Steam Beta Opt-in** | Modern Steam UI hook conflicts with legacy engine | Bypasses launch-day hooks entirely |
| **Client Restart** | Unapplied Steam system updates | Refreshes local runtime environment |

<Callout type="warning" title="Important Step">

### Step-by-Step Steam Client Configuration

<Steps>
  <Step num="1" title="Verify Game Files">
    Open Steam, navigate to your Library, and right-click on Call of Duty: Black Ops II (Singleplayer, Multiplayer, or Zombies). Select **Properties**, go to the **Installed Files** (or Local Files) tab, and click **Verify integrity of game files**. Let the process finish and download any missing components.
  </Step>
  <Step num="2" title="Opt into Steam Client Beta">
    Click **Steam** in the top-left corner of the client, choose **Settings**, and head to the **Interface** tab. Find **Client Beta Participation**, change it to **Steam Beta Update**, and let the client restart.
  </Step>
  <Step num="3" title="Re-Verify Post-Update">
    Once the Steam client updates and restarts in Beta mode, perform the file verification step from Step 1 once more to ensure the game files align with the new client hooks.
  </Step>
</Steps>

<Section icon={Swords} title="Reinstalling DirectX and Visual C++ Redistributables">

Legacy titles require precise runtime libraries that modern Windows 10 and Windows 11 installations often lack or override with newer, incompatible versions. Manually deploying the redistributables packaged directly with the game files resolves deep-level system execution mismatches.

| Package Name | File Path in Directory | Function |
| :--- | :--- | :--- |
| **vcredist_x86.exe** | `...\Call of Duty Black Ops II\redist\` | Visual C++ Runtime libraries |
| **DXSETUP.exe** | `...\Call of Duty Black Ops II\redist\DirectX\` | Legacy DirectX 9.0c API hooks |

<Callout type="tip" title="Pro Tip">
Always run these installer executables as an Administrator. Right-click the setup file and select **Run as administrator** to prevent Windows User Account Control from blocking critical system directory writes.
</Callout>

### Manual Dependency Installation Guide

<Steps>
  <Step num="1" title="Navigate to Local Files">
    Right-click the game in your Steam Library, select **Manage**, and click **Browse local files**. This opens your installation directory.
  </Step>
  <Step num="2" title="Install Visual C++ Redistributable">
    Open the `redist` folder. Locate and run `vcredist_x86.exe`. Choose the repair or install option, complete the setup, and close the window.
  </Step>
  <Step num="3" title="Run DirectX Setup">
    Inside the same `redist` folder, open the `DirectX` folder. Scroll down to find `DXSETUP.exe`, run it, accept the agreement, and let it complete the legacy API installation.
  </Step>
</Steps>

</Section>

<Section icon={Target} title="Compatibility and Overlay Adjustments">

The Call of Duty: Black Ops II engine frequently conflicts with modern operating system schedulers and background overlay programs. Adjusting compatibility modes and closing monitoring software prevents memory access violations during startup.

<CardGrid cols={3}>
  <Card title="Windows Compatibility">
    Force the game executables (`t6mp.exe`, `t6sp.exe`, or `t6zm.exe`) to run in **Windows 7** or **Windows 8** compatibility mode via properties.
  </Card>
  <Card title="Disable Overlays">
    Turn off GeForce Experience, Discord, and Steam overlays. These programs attempt to inject code into the rendering pipeline, causing instant crashes.
  </Card>
  <Card title="Close Hardware Monitors">
    Exit utility software such as MSI Afterburner and RivaTuner Statistics Server (RTSS) before launching the game executable.
  </Card>
</CardGrid>

<Callout type="success" title="Crash Resolved">
